# Deep Signal

An indoor dive-navigation game. No GPS — players navigate five dive sites by
**compass bearing** and **counted fin-strokes**, logging each find with a code
word from a physical plaque, then surface where they started.

Static site, no backend. Installable as a PWA and fully playable offline.

## Files

| | |
|---|---|
| `index.html` | the whole app — route, content, compass, pedometer, UI |
| `manifest.webmanifest`, `sw.js` | PWA install + offline shell |
| `icons/` | generated by `make_icons.py` |
| `make_plaques.py` | builds the five A5 QR plaques → `plaques/deep-signal-plaques.pdf` |

## The route

Six logged stops in a closed loop out of and back to the VIZ booth (440).
The order is enforced in the app, so plaques can be hung in any physical
arrangement.

| # | code | stop | leg in | bearing | dist |
|---|---|---|---|---|---|
| — | — | 440 VIZ (start) | — | — | — |
| 1 | `WRECK` | Historical Diving Society 340 | 440 → 340 | 323.9° | 11.9 m |
| 2 | `CORAL` | Tech Stage | 340 → tech | 323.9° | 14.0 m |
| 3 | `DRIFT` | Forster Dive Center 134 | tech → 134 | 233.9° | 8.7 m |
| 4 | `ABYSS` | south-east wall | 134 → wall | 143.9° | 38.6 m |
| 5 | `CAVERN` | Royal Australian Navy 512 | wall → 512 | 233.9° | 14.7 m |
| 6 | `PELAGIC` | behind the Main Stage | 512 → stage | 53.9° | 28.6 m |
| — | — | 440 VIZ (finish) | stage → 440 | 323.9° | 10.3 m |

Bearings are **magnetic** degrees, measured off `map&itinerary.pdf`: booth
modules give the scale (3 m = 68.3 px at 8× render) and the compass rose gives
north (page angle 77.8°).

The rose points **magnetic** north, not true. Sydney's declination is ~12° E,
so `true = magnetic + 12` — which places true north at page angle 89.8°,
i.e. straight up the page, as you'd expect a hall plan to be drawn. That
0.2° residual is the independent check that the scale and rose readings are
sound.

The floor grid sits **24.1° off true cardinal** (36.1° off magnetic — the two
differ by the declination), leaving only two aisle directions, 323.9/143.9 and
53.9/233.9, so every leg is a single straight run down one of them. Total
126.8 m; the vector loop closes to 5.7 m, inside the ~9.6 m of QR gaps the
drawing doesn't cover.

Raw device heading is mapped onto that frame by one offset captured at
calibration (`CAL_BEARING`), while the player points along leg 1. **A constant
frame mismatch cancels out** — only the differences between leg bearings
survive the offset — so magnetic-vs-true only affects the *skip calibration*
path, where an offset of 0 trusts the handset's compass directly. Storing
magnetic is deliberate: that's the frame handsets report in.

## URL parameters

**`?site=CODE`** — what the plaque QR codes encode. On load the code is held
until the player is actually navigating, then checked against the site they
are currently on:

* matches the current site → the code-gate is auto-filled and confirmed, any
  remaining legs are closed out (this is the dead-reckoning drift reset), and
  the quiz opens
* matches a site further along → refused, with a note pointing back to the
  current site
* matches an already-logged site → refused as already logged

The code is stripped from the URL on load so a reload can't replay it.

**`?dev=1`** — desktop harness. Adds a bar with a heading slider, `face`
(snap to the current leg's target bearing), `step` / `step ×10`, `walk leg`
and `code`. Keyboard: `←`/`→` turn 5°, `space` steps, `f` faces the bearing.
It feeds the same `heading` variable and `step()` entry point the real
sensors use, so nothing past the sensor boundary is stubbed.

The two combine: `?dev=1&site=WRECK`.

## Regenerating the plaques

```sh
pip install qrcode pillow
python make_icons.py
python make_plaques.py --base https://<your-pages-url>/
python make_plaques.py --theme light      # toner-friendly variant
```

`make_plaques.py` cross-checks its site codes against `index.html` and exits
if they have drifted apart.

## Deploying

GitHub Pages, `main` branch, root. Bump `CACHE_VERSION` in `sw.js` on every
deploy or returning players keep the cached shell.
