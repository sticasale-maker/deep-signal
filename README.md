# Deep Signal

An indoor dive-navigation game. No GPS ‚Äî players navigate five dive sites by
**compass bearing** and **counted fin-strokes**, logging each find with a code
word from a physical plaque, then surface where they started.

Static site, no backend. Installable as a PWA and fully playable offline.

| | |
|---|---|
| `index.html` | the whole app ‚Äî route, content, compass, pedometer, UI |

## The route

Six logged stops in a closed loop out of and back to the VIZ booth (440).
The order is enforced in the app, so plaques can be hung in any physical
arrangement.

| # | code | stop | leg in | bearing | dist |
|---|---|---|---|---|---|
| ‚Äî | ‚Äî | 440 VIZ (start) | ‚Äî | ‚Äî | ‚Äî |
| 1 | `WRECK` | Historical Diving Society 340 | 440 ‚Üí 340 | 323.9¬∞ | 11.9 m |
| 2 | `CORAL` | Tech Stage | 340 ‚Üí tech | 323.9¬∞ | 14.0 m |
| 3 | `DRIFT` | Forster Dive Center 134 | tech ‚Üí 134 | 233.9¬∞ | 8.7 m |
| 4 | `ABYSS` | south-east wall | 134 ‚Üí wall | 143.9¬∞ | 38.6 m |
| 5 | `CAVERN` | Royal Australian Navy 512 | wall ‚Üí 512 | 233.9¬∞ | 14.7 m |
| 6 | `PELAGIC` | behind the Main Stage | 512 ‚Üí stage | 53.9¬∞ | 28.6 m |
| ‚Äî | ‚Äî | 440 VIZ (finish) | stage ‚Üí 440 | 323.9¬∞ | 10.3 m |

Bearings are **true** degrees, measured off `map&itinerary.pdf`: booth
modules give the scale (3 m = 68.3 px at 8√ó render) and the compass rose gives
north (page angle 77.8¬∞). The floor grid is rotated ~11.7¬∞ from cardinal, so
only two aisle directions exist ‚Äî 323.9/143.9 and 53.9/233.9 ‚Äî and every leg
is a single straight run down one of them. Total 126.8 m; the vector loop
closes to 5.7 m, which is inside the ~9.6 m of QR gaps the drawing doesn't
cover.

Raw device heading is converted to true bearing by one offset captured at
calibration (`CAL_BEARING`), while the player points along leg 1.

## URL parameters

**`?site=CODE`** ‚Äî what the plaque QR codes encode. On load the code is held
until the player is actually navigating, then checked against the site they
are currently on:

* matches the current site ‚Üí the code-gate is auto-filled and confirmed, any
  remaining legs are closed out (this is the dead-reckoning drift reset), and
  the quiz opens
* matches a site further along ‚Üí refused, with a note pointing back to the
  current site
* matches an already-logged site ‚Üí refused as already logged

The code is stripped from the URL on load so a reload can't replay it.

**`?dev=1`** ‚Äî desktop harness. Adds a bar with a heading slider, `face`
(snap to the current leg's target bearing), `step` / `step √ó10`, `walk leg`
and `code`. Keyboard: `‚Üê`/`‚Üí` turn 5¬∞, `space` steps, `f` faces the bearing.
It feeds the same `heading` variable and `step()` entry point the real
sensors use, so nothing past the sensor boundary is stubbed.

The two combine: `?dev=1&site=WRECK`.

## Regenerating the plaques

```sh
pip install qrcode pillow
python make_icons.py
python make_plaques.py --base https://<your-pages-url>/
python make_plaques.py --theme light      # toner-friendly variant
```

`make_plaques.py` cross-checks its site codes against `index.html` and exits
if they have drifted apart.

## Deploying

GitHub Pages, `main` branch, root. Bump `CACHE_VERSION` in `sw.js` on every
deploy or returning players keep the cached shell.
